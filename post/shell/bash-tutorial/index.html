1. 文件或目录测试运算符

参数 意义
-b file 检测文件是否是块设备文件
-c file 检测文件是否是字符设备文件
-d file 检测文件是否是目录
-f file 检测文件是否是普通文件
-g file 检测文件是否设置了 SGID 位
-k file 检测文件是否设置了粘着位(Sticky Bit)
-p file 检测文件是否是有名管道
-u file 检测文件是否设置了 SUID 位
-r file 检测文件是否可读
-w file 检测文件是否可写
-x file 检测文件是否可执行
-s file 检测文件是否为空（文件大小是否大于0）
-e file 检测文件（包括目录）是否存在

e.g:

##判断文件是否存在
if [ ! -e test.sh ];then    #--> ! 为否定，注意空格字符
    echo "test.sh 文件的确不存在！"
else
    echo "test.sh 文件已存在！"
fi

2. $n参数传递意义

参数 意义
$# 传递到脚本的参数个数
$* 以一个单字符串显示所有向脚本传递的参数,所有参数输出就是一个字符串
$$ 脚本运行的当前进程ID号
$! 后台运行的最后一个进程的ID号
$@ 与$*相同，但是使用时加引号，并在引号中返回每个参数
$- 显示Shell使用的当前选项，与set命令功能相同。
$? 显示最后命令的退出状态。0表示没有错误，其他任何值表明有错误

e.g:

kei@kei-desktop:~/Work/bash$ cat test.sh 
#!/bin/bash
echo $#      #--> 显示传入的参数个数
kei@kei-desktop:~/Work/bash$ ./test.sh var1
1
kei@kei-desktop:~/Work/bash$ ./test.sh var1 var2
2
kei@kei-desktop:~/Work/bash$ ./test.sh var1 var2 var3
3

kei@kei-desktop:~/Work/bash$ cat test.sh 
#!/bin/bash
echo $*      #--> 以单行字符串形式显出 输入的参数
echo $@
kei@kei-desktop:~/Work/bash$ ./test.sh var1 var2 var3
var1 var2 var3

kei@kei-desktop:~/Work/bash$ cat test.sh 
#!/bin/bash
echo $$      #--> 显示当前脚本进程ID
echo $?      #--> 显示前一行的命令是否执行成功，0没有错。
ls -e ./ >/dev/null 2>1     #--> ls 没有参数 -e,所以此行命令必定出错
echo $?
kei@kei-desktop:~/Work/bash$ ./test.sh var1 var2 var3
1618
0
2

3. 算术及关系符号运用

  • 1.算术符号：加+,减-,*,除/,取模%,=,!=,==

注意：以上算术运算符，左右必须带空格。

e.g:

#!/bin/bash
A=10
B=20
C=$(expr $A + $B)   ＃注意，取模及四则运算，必须采用命令expr，或let来执行
echo $C   #--> 此处输出结果为30
＃若不带空格
D=$(expr $A+$B)
echo $D   #-->此处输出结果为10＋20，当字符串拼接处理
  • 2.关系符号：等于-eq,小于-lt,大于-gt,小于且等于-le,大于且等于-ge,不等于-ne

[ ] 中括号 为条件运算区域，不得跟其它字符连接 eg:

#!/bin/bash
A=10
B=20
if [ $A -lt $B ]
    then echo "$A 小于 $B"
else
    echo "$A 不小于 $B"
fi
# --> 结果输出 10 不小于 20.

4. 重定向输入及输出stdin(0),stdout(1),stderr(2)

  • 标准重定向输入文件(stdin),stdin其文件描述符为0
  • 标准重定向输出文件（stdout),stdout其文件描述符为1
  • 标准错误文件（stderr），stderr其文件描述符为2
  • >,<覆盖；>>,<< 追加
  • 2>&1 意义为 stdout及stderr均合并
  • EOF 特殊的文档标记重定向，多用于交互式输入

示例：

a.标准重定向输入出到文件(注意>为覆盖写入))
  命令格式：command > file
  ls ~/Work/bash > ~/out.txt

b.希望将错误stderr(2) 重定向输出到文件 error.log中：
  命令格式：command 2> file
  ls -e ~/Work/bash 2> ~/error.log

c.若希望将 stdout(1) 及 stderr(2) 合并后 重定向到文件中：
  命令格式：command > file 2>&1 或者 command >> file 2>&1
  ls -e ~/Work/bash > ~/error.log 2>&1

d.若希望将 stdout(1) 及 stderr(2) 信息均屏蔽，可通过 输出空块文件操作：
  命令格式：command > /dev/null 2>&1

e.
 cat <<EOF >> ~/Work/bash/out.txt
 > hello world
 > EOF

＃--> EOF为标记文档的区域标志，如上所示，EOF 与EOF间就是文档内容，把此内容重定向追加到out.txt中去
